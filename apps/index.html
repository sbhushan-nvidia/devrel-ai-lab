📋 Pre-Work (Complete Before the Session)

⚠️ Complete these steps at least 1 day before the lab

The lab session is only 30 minutes — there's no time for setup during the session. Budget ~30 minutes for pre-work.

Detailed guide: Full Pre-Work Guide on GitHub

1

Set up your DGX Horde pod

Request a pod, SSH in, install OpenClaw, and start the gateway.

curl -fsSL https://get.openclaw.ai | bash
openclaw gateway start
2

Create your Telegram bot

Open Telegram → search @BotFather → send /newbot → follow prompts → copy the bot token.

Then on your pod:

openclaw configure --section telegram
# Paste your bot token when prompted
openclaw gateway restart
3

Set up GitHub

You need: a GitHub account, a personal access token, and a public repo.

  1. Create a fine-grained token with Contents + Pages + Administration (all Read+Write)
  2. Create a public repo at github.com/new called my-event-tracker
  3. Enable GitHub Pages: repo → Settings → Pages → Source: GitHub Actions
  4. On your pod:
    gh auth login
# Choose HTTPS, paste your token
git config --global user.name "Your Name"
git config --global user.email "you@nvidia.com"
4

Verify everything

Run this on your pod:

curl -fsSL https://sbhushan-nvidia.github.io/devrel-ai-lab/verify.sh | bash

All green = you're ready. Send a screenshot of the results to Shashi.

📱 Test Your Bot

Send your bot a message in Telegram: Hello! Are you there? — if it replies, you're good to go.

🏗️ During the Session

Step 1: Pick your style

Choose an accent color for your app:

Orange #FF6F00

Teal #1ABC9C

Step 2: Paste the prompt

Copy this into your Telegram bot. Replace the 3 fields in [BRACKETS] with your info:

Build me a single-file web app called "[YOUR NAME]'s Event Tracker".

Requirements:
- One HTML file, all CSS and JS inline, no external dependencies
- NVIDIA dark theme: background #1A1A1A, primary green #76B900, accent [YOUR COLOR]
- Responsive, clean, professional design

Pages (use hash routing):

1. DASHBOARD (#dashboard) — default page
   - 4 metric cards: Total Events, Total Attendees, Avg Rating, Upcoming Count
   - Bar chart of events per month (use canvas)
   - List of 5 most recent events

2. EVENTS (#events)
   - Searchable, filterable list of all events
   - Each event: name, type, date, location, attendee count, status badge
   - Add/edit event form (name, type, date, location, description)
   - Types: conference, workshop, meetup, webinar, office hours

3. EVENT DETAIL (#event/:id)
   - Event info header with status
   - Attendee table (name, email, company, RSVP, checked-in toggle)
   - Add attendee form
   - Feedback section: avg rating display + add feedback (1-5 stars, comment)

4. SETTINGS (#settings)
   - Display name: [YOUR NAME]
   - Team: [YOUR TEAM]
   - Export/import data as JSON
   - Clear all data (with confirmation)

Data: localStorage. Seed with 5 demo events so it's not empty.
Make it look polished — I'll actually use this.

Save the file as index.html in my workspace.

Step 3: Wait (~1-2 min)

Your bot will generate the app. When it's done, it will tell you.

Step 4: Deploy it

Tell your bot:

Deploy my app to GitHub Pages. Here's what to do:
1. Clone my repo [YOUR-GITHUB-USERNAME]/my-event-tracker
2. Create a .github/workflows/deploy.yml that deploys to GitHub Pages
3. Put my index.html at the root
4. Commit and push to main
Use the GitHub token configured via gh auth.

Your app will be live at: https://[your-github-username].github.io/my-event-tracker/

Step 5: Customize!

This is the fun part. Talk to your bot to change anything:

💡 Customization Ideas

Easy One message each:

  • "Add a notes field to events"
  • "Change event types to: conference, hackathon, demo day, webinar, meetup"
  • "Add a budget field to events"
  • "Add a countdown to the next upcoming event on the dashboard"
  • "Make the status badges more colorful"

Medium A bit more complex:

  • "Add a pie chart showing events by type on the dashboard"
  • "Add CSV export for the attendee list"
  • "Add a search bar that searches across all events and attendees"
  • "Add a dark/light mode toggle"
  • "Show a monthly calendar view of events"

Advanced Push the limits:

  • "Add an email button that opens mailto: with all attendee emails"
  • "Add tags to events and let me filter by tag"
  • "Add a 'duplicate event' button that copies an event as a template"
  • "Add a timeline view showing events on a horizontal timeline"
  • "Add print-friendly styling for the event detail page"

After each change, tell the bot to commit and push the updated file to your repo.

🆘 If Something Goes Wrong

App has errors?

The app has an error. Please fix it and make sure it loads correctly.

Styling looks off?

Fix the styling. Use dark background #1A1A1A, green #76B900 headers, and clean modern design.

Want to start over?

Start fresh. Rebuild the entire app from scratch using the same requirements.

Bot is confused?

Ignore previous errors. Generate a complete single HTML file with [describe what you want].
